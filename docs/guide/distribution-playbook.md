# Distribution playbook

Use this playbook to promote God Kit by adding value first.

## Channel principles

- Lead with technical insight, not promotion.
- Use one practical link per post/thread.
- Tailor message to each community context.

## Awesome list submission copy

### awesome-vue candidate

`god-kit` - Vue 3 and Nuxt 4 admin UI kit with typed components, semantic design tokens, and a migration bridge from Vuetify.

### awesome-nuxt candidate

`god-kit` - Admin dashboard component library for Nuxt 4 with Vue 3 primitives, token-based theming, and migration-friendly CSS bridge.

## Reddit post template (educational)

Title idea:

`What we learned building a Vue 3 + Nuxt 4 admin UI kit (tokens, a11y, migration)`

Body skeleton:

1. Problem statement (consistency, migration pressure, dashboard complexity).
2. Technical decisions (tokens, typed exports, a11y test coverage).
3. Tradeoffs and what did not work.
4. Link to docs/demo at the end.

## Discord showcase/help template

Short format:

`Built a Vue 3/Nuxt 4 admin UI kit focused on forms + navigation + migration from Vuetify. Happy to share implementation details on token structure and a11y testing if useful.`

## Pre-post checklist

- Verify install instructions from a clean project.
- Ensure demo/docs links are live.
- Confirm release tag and changelog are current.
- Ask: "Does this message teach something even without clicking?"
